# Composio Tool Dependency Graph Approach

## Goal

Build a dependency graph for the `github` and `googlesuper` Composio toolkits.

An edge `A -> B` means:

> Tool A can help obtain information needed before Tool B can be executed safely or correctly.

Example:

```text
GOOGLESUPER_ACL_LIST -> GOOGLESUPER_ACL_DELETE
```

because `ACL_DELETE` needs `rule_id`, and `ACL_LIST` can enumerate ACL rules and expose rule IDs.

## Why This Is Hybrid

The raw Composio output schemas are generic:

```text
data
error
successful
```

So exact output-to-input schema matching is not enough. The graph builder therefore uses:

1. deterministic resource/candidate scoring
2. LLM adjudication over the top candidates
3. strict validation and conservative fallback

The LLM is not allowed to invent tools. It only chooses from candidates generated by rules.

## Pipeline

### 1. Fetch and normalize tools

```bash
npm run inventory
```

Writes:

```text
data/raw/*.json
data/normalized/*.json
data/summary/tool_inventory.md
```

### 2. Build candidate precursor sets

```bash
npm run graph:candidates
```

Writes:

```text
data/candidates/all_candidates.json
```

Each target tool gets up to 40 likely precursor candidates, scored by:

- shared toolkit
- matching canonical resources
- field/input mentions
- producer verbs like list/search/find/fetch/get/create
- same object family
- penalty for destructive source tools

### 3. Run LLM adjudication

The shipped graph can be built deterministically with:

```bash
npm run graph:build
```

This uses the resource/candidate layer to produce high-confidence edges quickly, and merges any successful LLM decisions from `data/llm/decisions/`.

Optional LLM adjudication can be run with:

```bash
npm run graph:infer
```

Useful options:

```bash
npm run graph:infer -- --limit=50 --concurrency=4
npm run graph:infer -- --offset=50 --limit=50 --concurrency=4
npm run graph:infer -- --max-candidates=30 --concurrency=8
```

Environment variables:

```text
OPENROUTER_API_KEY=single_key
OPENROUTER_API_KEYS=key1,key2,key3
```

The default model is:

```text
stepfun/step-3.5-flash
```

The run is resumable. Completed decisions are cached in:

```text
data/llm/decisions/
```

Errors are written to:

```text
data/llm/errors/
```

### 4. Render visualization

```bash
npm run graph:render
```

Writes:

```text
dependency_graph.html
```

## Accuracy Rules

An edge is accepted only if it:

- points from an existing candidate tool
- points to the target tool
- is not a self-edge
- names at least one satisfied input/prerequisite
- has confidence above the threshold
- includes a reason

If the LLM fails to return valid JSON, the system conservatively records no edges for that target rather than hallucinating.
